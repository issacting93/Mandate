01
Design Principles
DOCTRINE // WHAT THIS LANGUAGE STANDS FOR

Mandate is a civic governance simulation. Its visual language is a declassified ops document — structured, serious, traceable. Every design decision should feel like it belongs in a printed briefing sheet handed to a new mayor on their first day.

01
Document Over Interface
The screen reads as a physical artifact — a protocol printout, a field brief, a government manifest. UI chrome is minimized. Structure comes from typography and rules, not from rounded cards and drop shadows.
02
Restraint as Authority
Red is used sparingly and earns its meaning. It signals active state, critical action, and the brand itself. When red appears, something matters. An interface that uses red for everything signals nothing.
03
Type Does the Work
Bebas Neue carries structural weight — section numbers, card titles, budget figures, the logo. Space Mono carries data weight — labels, codes, metadata. These two voices in dialogue create the entire personality of the screen.
04
Structure Is Information
Section numbers (01, 02) are earned by actual sequence, not decoration. Corner brackets mark targets. Barcodes signal physical document provenance. Every structural element encodes something true about the game mechanic it serves.
05
Faction Color, Not UI Color
The five faction colors exist only to identify political alignment on field briefing cards. They are never used for general UI states. The UI palette is black, red, and warm off-white — nothing else.
06
Ambient World-Building
The watermark, the classified stamp, the "Directive //" sub-labels — these are subliminal signals that ground the player in the fiction. They should be felt, not read. Opacity and placement make them furniture, not foreground.

04
Spacing & Grid
LAYOUT // STRUCTURE BEFORE DECORATION

The layout is a two-column document grid: a wide main field for directives and cards, a narrow sidebar for budget. Columns are separated by a single 1px rule — no gap, no shadow, no elevation.

Main field — directives, slots, field briefings

Border Radius

Zero. The Mandate language uses no border radius anywhere. Rectangles only. Rounded corners signal consumer softness; this is a government protocol interface.

Microdetail
TEXTURE // WHAT MAKES IT FEEL PHYSICAL
Watermark
WK 01 / 48
Content sits above at full opacity

The week counter in Bebas Neue at 3% opacity bleeds behind all layout content. Never interactive, never hoverable. Positioned center of the main panel. It is felt as spatial context, not read as information.

Classified Stamp
Classified · Mayoral Eyes Only

Bottom-right of main panel. Rotated -2.5°. Red at ~18% opacity. Communicates fictional authority without competing with any functional element. This is pure world-building texture.

Corner Brackets
Empty
2

7×7px L-shaped corner brackets rendered as CSS borders. Empty state: bracket color only, ghost number, dashed or solid rule border. Filled state: border activates to red, number turns red.

Barcode Strip

CSS-rendered barcode using variable-width spans at 10% opacity. Placed at the base of each field briefing card. Reference to physical document provenance — the card as a printed asset from city hall.

07
Do & Don't
COMPLIANCE // COMMON VIOLATIONS
✓ Do
Use red only for the logo, section numbers, corner brackets, active slot state, and primary CTA hover. Red earns attention by being rare.
✗ Don't
Use red for decorative accents, general highlights, or faction indicators. This dilutes the signal and breaks the scarcity that gives red its meaning.
✓ Do
Use 0px border-radius everywhere. All elements are rectangular — cards, slots, buttons, budget bars, dept rows.
✗ Don't
Round any corner. Border radius immediately reads as consumer UI and breaks the document protocol aesthetic.
✓ Do
Keep all text UPPERCASE only for eyebrows, labels, and directives — short strings at 7–10px. Body text and card intel are mixed case.
✗ Don't
Set long body text in uppercase. It destroys legibility and makes everything feel like a screaming warning label.
✓ Do
Maintain a uniform card grid with consistent min-height. Use a ghost card to complete incomplete grid rows. Structural consistency is a form of authority.
✗ Don't
Mix grid column counts across card rows (e.g. 3 cards on top, 2 on bottom). Inconsistent sizing breaks the protocol read and feels undisciplined.
✓ Do
Use faction color only on the 2px top border of field briefing cards. The color is identification, not decoration.
✗ Don't
Use faction colors for backgrounds, text, or any UI state. This confuses political identity with interface meaning.
✓ Do
Keep the watermark at 3% opacity and the classified stamp at ~18% opacity. They exist at the edge of perception.
✗ Don't
Increase the opacity of decorative world-building elements. The moment they become readable, they become noise.
